Download

下载与依赖

发布包提供两种模式，按场景选择。推荐优先下载便携版。

Portable 便携版

自带依赖，开箱即用。适合多数用户。

  • 体积更大，但无需额外运行库
  • 适合快速部署或离线使用

System 依赖版

不包含运行库，体积更小。

  • 需要 Windows App SDK Runtime
  • 需要 VC++ 运行库与 WebView2 Runtime

Releases 地址：https://github.com/RE-TikaRa/FileLocksmith/releases

Usage

运行方式与 CLI

可以通过 GUI 管理、右键菜单或命令行使用。

直接启动

运行主程序进入管理界面，统一管理注册状态与显示方式。

右键菜单

从资源管理器进入扫描/解锁界面，定位占用并结束进程。

CLI 工具

FileLocksmithCLI.exe 支持 JSON 输出、等待解锁与超时控制。

FileLocksmithCLI.exe [选项] <路径1> [路径2] ...
选项:
--kill    结束占用文件的进程
--json    以 JSON 格式输出结果
--wait    等待文件解锁
--timeout  --wait 的超时（毫秒）
--help    显示帮助

配置与数据位置

  • 根目录：%LocalAppData%\ALp_Studio\FileLocksmith
  • 设置文件：file-locksmith-settings.json
  • 日志：Logs\log.log / Logs\Log_YYYY-MM-DD.log

状态与策略

  • 组策略：HKLM\Software\Policies\FileLocksmith
  • 注册标记：HKCU\Software\FileLocksmith

FAQ

常见问题

快速回答下载、依赖与使用场景的关键问题。

便携版与 System 依赖版有什么区别？

便携版自带运行库，体积更大但开箱即用；System 版更轻量，需要系统已安装运行时依赖。

为什么需要提升权限？

涉及注册/卸载右键菜单与查看系统进程时需要管理员权限，以确保操作可生效。

设置与日志存放在哪里？

默认位于 %LocalAppData%\\ALp_Studio\\FileLocksmith，可在“配置与数据位置”中查看详细路径。

CLI 适合什么场景？

适合自动化脚本或批量解锁场景，支持 JSON 输出、等待解锁与超时控制。
